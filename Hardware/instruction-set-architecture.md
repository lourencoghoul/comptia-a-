##  ISA (Instruction Set Architecture)

ISA (Instruction Set Architecture) is the set of instructions that a processor can understand and execute. It defines how software communicates with hardware.

---

##  What it includes

* Supported instructions (addition, subtraction, movement, etc.)
* Data types
* Registers
* Memory addressing modes

---

##  Examples

* x86
* x86-64
* ARM

---

##  Note

ISA acts as a bridge between software and hardware, allowing programs to run on a specific type of processor.


##  x86-32 and x86-64 (Processor Architectures)

x86-32 and x86-64 are processor architectures (ISA), meaning a set of rules that define how the CPU works, processes data, and executes instructions.
They are also related to Assembly language, which is a human-readable representation of computer instructions.

---

##  x86 (32-bit)

x86 is a 32-bit architecture and is older.

* Supports up to **4 GB of RAM**
* Uses 32-bit registers
* Related to Assembly language
* More limited in performance and capacity

---

##  x86-64 (64-bit)

x86-64 is the evolution of the x86 architecture, with 64-bit support.

* Supports **more than 4 GB of RAM**
* Uses 64-bit registers
* Better performance and higher processing capacity
* Also uses Assembly language

##  ARM (Processor Architecture)

ARM is a processor architecture (ISA), meaning a set of rules that defines how the CPU works, processes data, and executes instructions.
It is also related to Assembly language, which represents computer instructions in a human-readable form.

---

##  ARM

ARM is a modern architecture focused on efficiency and low power consumption.

* Uses **RISC (Reduced Instruction Set Computing)**
* Low power consumption
* Generates less heat
* High efficiency
* Widely used in mobile and embedded devices

---

##  Note

ARM is different from x86/x86-64 because it prioritizes energy efficiency, making it ideal for smartphones, tablets, and portable devices.

## Example
Every CPU uses an ISA (Instruction Set Architecture), but there is no single universal ISA. Each processor is designed to work with a specific ISA, such as **Intel and AMD processors** using x86/x86-64, and **smartphones** using ARM. Programs must be compatible with the CPU’s ISA to run properly.
